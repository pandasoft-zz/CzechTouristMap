# CzechTouristMap

Open-source Mapsforge render theme for Czech Republic tourist maps.

## Motivation

The goal of this project is to create a freely available map style for offline maps (Mapsforge) that closely resembles the visual style of Czech tourist maps — the kind that hikers know from paper maps published by KČT, SHOKart or mapy.cz.

This style is quite unique. Czech tourist maps have their own distinct visual language, different from maps used abroad. This is especially true for:

- **Trail markings** — the system of colored hiking routes (red, blue, green, yellow) with their characteristic symbols originates from the Czech Republic and is specific to the Central European region
- **Cartographic symbols** — colors and symbols follow the ČÚZK standard (Czech Office for Surveying, Mapping and Cadastre), specifically the Signature Manual (SM5)
- **Visual style** — terrain representation, road markings, settlements and natural features follow the conventions of Czech topographic maps, which differ significantly from styles used in Western Europe or global mapping projects

The project was created as an open-source alternative to proprietary map styles and is intended for use in offline mapping applications (OruxMaps, Locus Map, AlpineQuest, etc.) with OpenAndroMaps map data.

## Technology

- **Mapsforge** — vector map format and render engine
- **XSLT** — templates for generating the XML render theme
- **OpenHiking** — modular XSLT base (GPL-3.0)
- **Docker** — reproducible build system

## Running locally (tile server)

The tile server renders map tiles from a Mapsforge `.map` file.
The map file is stored in a Docker volume and populated from a GHCR image on first start.

### First-time setup

1. Push the map image to GHCR (one-time, or after updating the map file):
   ```bash
   ./scripts/push-map-image.sh <github-owner>
   ```

2. Start the stack:
   ```bash
   docker compose up
   ```
   On first run, `map-init` copies `czech-republic.map` from the GHCR image into the
   `map-data` volume. Subsequent starts skip this step automatically.

3. Open the map viewer at `http://localhost:8080`.

### Updating the map file

1. Replace your `.map` file and update `config/map-version.txt`.
2. Re-run `push-map-image.sh` to publish the new image.
3. Re-create the volume to fetch the new file:
   ```bash
   docker compose down -v
   docker compose up
   ```

### Manual volume population (no GHCR access)

```bash
docker volume create maprender_map-data
docker run --rm \
  -v "$PWD/maps/czech-republic.map:/src/czech-republic.map:ro" \
  -v maprender_map-data:/export \
  busybox cp /src/czech-republic.map /export/czech-republic.map
docker compose up --no-deps tile-server
```
